# A Comparative Semantic Audit of Python Code Generation

## Overview

This repository provides a high-fidelity orchestration framework designed to audit the security posture of Python code generated by Large Language Models (LLMs). By leveraging **CodeQL** for static analysis and mapping results to the **OWASP Top 10 (2025)**, this tool quantifies the propensity of different models to introduce Common Weakness Enumerations (CWEs) into their outputs.

The framework is built with a **"Security-First"** philosophy, utilizing bit-level integrity validation and professional telemetry to ensure the reproducibility and validity of scientific experimental results.

---

## Technical Architecture

### Core Pipeline Modules

* **`main.py`**: The central orchestrator. Manages the execution lifecycle, session resumption logic, and the UI lifecycle via a centralized Telemetry system.
* **`src/llm/`**: Contains model-specific API adapters (Anthropic, Gemini, Mistral, Meta) and the code generation pipeline logic.
* **`src/scanners/codeql.py`**: The analytical engine. Automates CodeQL database initialization, static analysis execution using the `python-security-and-quality` suite, and SARIF report generation.
* **`src/core/integrity.py`**: The security gateway. Implements SHA-256 bitstream validation for both the framework's source code (Master Baseline) and the LLM-generated artifacts.
* **`src/analysis/stats.py`**: The scientific engine. Performs automated **ANOVA**, **Chi-Squared**, and **Tukey HSD** post-hoc testing to determine statistical significance between model performances.

### Configuration & Telemetry

* **`config.py`**: Centralized manifest for API hyperparameters, directory maps, and the **Telemetry** class, which drives the Rich-based CLI (Logs, Panels, and Live Status Spinners).
* **`master_hashes.json`**: The cryptographic baseline used to detect source code drift or unauthorized modifications during an audit run.
* **`/session-logs`**: Persistent, timestamped audit trails containing terminal outputs (`tee`), raw JSONL logs, and integrity manifests.

---

## Technical Features

### 1. Cryptographic Chain of Custody

Every audit session begins and ends with an integrity check. The system validates its own source code against a "Golden Master" baseline to ensure no environmental drift has occurred during the experiment.

### 2. Standardized Security Mapping

Results are not merely raw CWE counts. The framework automatically maps CodeQL findings to the **OWASP 2025** categories, providing high-level semantic context for the vulnerabilities discovered.

### 3. Automated Post-Hoc Analysis

When a statistically significant difference is found between models ($p < 0.05$), the system automatically triggers a Tukey HSD test to identify the specific "difference makers," removing the need for manual data post-processing.

## Getting Started

### Prerequisites

* **CodeQL CLI**: Must be installed and available in the system `PATH`.
* **Python 3.10+**: Utilizes modern type-hinting and pipe operators.

### Installation

1. Clone the repository.
2. Install dependencies: `pip install -r requirements.txt`
3. Configure your `.env` file with the necessary LLM API keys.
4. Freeze your source code baseline: `python src/tools/freeze_hashes.py`

### Usage

To begin a full audit run:

```bash
python main.py

```

To resume a previous audit session, follow the on-screen prompts to select a recent session ID from the `Recent Audit Sessions` panel.
