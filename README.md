# A Comparative Semantic Audit of Python Code Generation
## Overview

This repository provides a high-fidelity orchestration framework designed to audit the security posture of Python code generated by Large Language Models (LLMs). By leveraging CodeQL for static analysis and mapping results to the OWASP Top 10 (2025), this tool quantifies the propensity of different models to introduce Common Weakness Enumerations (CWEs) into their outputs.

The framework is built with a "Security-First" philosophy, utilizing bit-level integrity validation to ensure the reproducibility and validity of experimental results.
Technical Architecture
## Core Modules

* `main.py`: The central orchestrator. It manages the execution lifecycle, from environment setup and API calling to triggering the analytical suite.

* `integrity_manager.py`: The security gateway. Implements SHA-256 bitstream validation for both the framework's source code and the LLM-generated artifacts.

* `audit_manager.py`: Data processing layer. Handles raw API response sanitization, SARIF report parsing, and multi-model logging.

* `stat_generation.py`: The scientific engine. Performs automated ANOVA and Tukey HSD post-hoc testing to determine statistical significance between model performances.

## Configuration & Data

* `config.py`: Centralized manifest for API hyperparameters (temperature, max tokens), directory maps, and OWASP category mappings.

* `master_hashes.json`: The "Golden Master" cryptographic baseline used to detect source code drift during execution.

* `/session-logs`: Persistent, timestamped audit trails containing terminal outputs, raw JSONL logs, and SARIF reports for every experimental run.

* `/seceval-dataset`: Localized storage for the foundational security-prompt dataset.
